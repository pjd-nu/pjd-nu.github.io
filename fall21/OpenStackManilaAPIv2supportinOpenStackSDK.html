Project Overview:

Background:  The OpenStack Shared File Systems service (codenamed "Manila") is a collection of micro services and associated components to provide self-service management of elastic file system storage infrastructure. With the Shared File Systems service, you can create POSIX compliant remote file systems for simultaneous read/write (RW) access from multiple clients. Manila was the first-of-its-kind to define the technology and add RESTful semantics to consuming shared storage in a reliable and scalable manner. This space now includes the Amazon Elastic File System (EFS), Azure Files and Google File System although these projects are proprietary. Users use Manila to provide read-write-many (RWX) storage to Virtual Machines, Bare Metal Compute Instances and Containers. With Manila, you could provision storage via 35+ storage technologies such as CephFS, GlusterFS, HDFS, ZFS, NetApp, GPFS, Dell EMC, etc over file system protocols such as NFS, CIFS, CephFS, MapRFS, HDFS and GlusterFS. The collective software is open source and hosted in the OpenStack Ecosystem (https://opendev.org).

You can read more about OpenStack Manila here: https://docs.openstack.org/manila/latest/

The contributor reference is here: https://docs.openstack.org/manila/latest/contributor/index.html

The REST API Specification is here: https://docs.openstack.org/api-ref/shared-file-system/

Project Specifics: python-openstacksdk is a Software Development Toolkit (SDK) written for python environments. It makes interacting with REST APIs exposed by OpenStack services a whole lot easier. As with any SDK, the aim of this project is to improve usability by providing consistent usage patterns and abstractions alongside operator tools and troubleshooting aids. Users of the SDK can use canned procedures to authenticate with any OpenStack cloud and manipulate cloud resources. Manila's version 2 (v2) API has growing support in the OpenStackSDK. There are still many "resources" (units of user facing objects that represent assets associated with Shared File Systems) that need to be exposed via the OpenStackSDK.

Students who pick this project can work with the OpenStack Manila team on completing these efforts, collaboratively. These projects will allow interns to understand cloud orchestration, building and maintaining APIs, using and building tools for declarative programming software such as Ansible.
