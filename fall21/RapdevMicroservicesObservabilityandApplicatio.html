Project Overview:

Background: As software engineering has evolved, application architecture shifted from a monolithic tech stack towards a more multi-layered, and resilient approach including microservices, cloud-native environments, and distributed systems. Consequently, application performance monitoring tools were developed to address the challenge of efficiently pinpointing the root cause of an issue in such complex systems.

Application Performance Management (APM) refers to monitoring the performance and availability of an application. This is typically done using APM software, a robust solution that helps engineers keep track of application and system metrics, such as resource consumption, response time, to ensure that the application is performing in a consistent manner. With APM, monitoring data is collected and displayed through customizable dashboards to provide information about when and where an issue may have occurred.

Project Specifics: Students will develop and deploy a cloud-native microservices application, such as a social network. The application should have a front end, business logic, and caching/storage. The application development should be automated and instrumented as early as possible. Using Agile methods, students will make gradual and iterative improvements to the application. The code will be in a Git repo on GitHub. GitHub Actions, Terraform, and Ansible will be used for automating the continuous testing, integration, deployment, and delivery.
