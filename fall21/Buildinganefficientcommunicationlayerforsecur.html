Project Overview:

Background: Cryptographically secure multi-party computation (MPC) [1, 2] allows mutually distrusting parties to compute arbitrary computations on their collective data while keeping their own private data siloed from each other and from external adversaries. MPC protocols achieve strong security via distributing random “shares” of the data across non-colluding entities (e.g., competing cloud providers) and guarantee that no single entity is able to reconstruct the original data. Compared to traditional “plaintext” computation, MPC requires exchanging more data between parties and, thus, its performance heavily relies on the communication layer.

 

Project Specifics: In this project students will work on Secrecy, a novel MPC framework for cloud-based secure data analytics. The goal of the project is to implement an efficient asynchronous communication layer for Secrecy using standing TCP connections between parties. As a second step, students will run a given Secrecy application on a Linux unikernel (UKL) prototype developed at BU to further speed up MPC using OS-level optimizations.

Some Technologies you will learn/use:

  • How to deploy applications in the cloud using VMs and OpenShift containers
  • How to run applications on UKL
  • How to profile the performance of distributed systems, such as Secrecy
  • How to implement synchronous and asynchronous communication using TCP
