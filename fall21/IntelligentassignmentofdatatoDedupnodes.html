Project Overview:

Background:  

Data is growing exponentially. It needs to be managed and stored on cloud-scale storage. Storage can be primary storage and secondary storage (backup). Data deduplication helps in reducing the physical footprint of storage and hence reducing the cost of storage. This is especially crucial in the secondary storage (backup) domain where similar data is backed up again and again. Compared to single node appliance storage, achieving high and balanced deduplication is a challenge in cloud-scale storage systems.

Project Specifics:

Deduplication is a widely used technique to reduce storage requirements by replacing redundant chunks of data with references.  For protection workloads, deduplication can reduce storage requirements to 1/55th of the typical space usage.  As storage systems grow to the cloud-scale, it is necessary to subdivide data and distribute it to deduplicating nodes to meet the capacity and throughput requirements of the cloud.  Once assigned to a node, deduplication will take place independently of other nodes to reduce coordination overheads.

 

In this project, we will explore ways to divide incoming files into large regions (~1MB-8MB) and assign regions to deduplication nodes.  Our goals are to achieve high levels of space savings as well as balanced use of ~1,000 deduplicating nodes.

 

 

Specifically, the project involves writing a storage simulator that will run on multiple nodes or as containers in an orchestration environment.  The simulator will process storage traces and experiment with different techniques to divide the data into regions and create an assignment for each region. Fingerprint is created using one of the hashing technique and sent to assigned node. Simulated dedup nodes will store the unique fingerprints into a distributed KV store which is used to deduplicate the data.

 

Deduplication space savings will be measured for each node and across the cluster and compute efficiency will be considered.
