Project Overview:

Background:  

High performance computing (HPC) and cloud technologies are increasingly coupled to accelerate the convergence of traditional HPC with new simulation, data analysis, machine-learning, and artificial intelligence approaches. While the HPC+cloud paradigm, or converged computing, is increasingly being adopted, several key mismatches between HPC and cloud technologies still preclude this paradigm from realizing its full potential.  IBM Research, Lawrence Livermore National Laboratory (LLNL), and Red Hat are working  together to address the mismatches and to bring full HPC scheduling awareness into Kubernetes, the de facto container orchestrator.

We are implementing KubeFlux, a plug-in scheduler for Kubernetes that uses Flux's advanced graph-based scheduler Fluxion as the module taking pod placements decision. Flux is a Resource and Job Management System designed for HPC. KubeFlux is open-source and it will be released as part of 4.10+ OpenShift release in 2022.

Project Specifics:

There are several aspects to be addressed when using third-party software to take scheduling decisions in a Kubernetes cluster. A very important one is state management due to resource sharing between Kubernetes' scheduler and the plug-in scheduler: Fluxion has its own internal state and resource management, and Fluxion's internal state of the cluster needs to be updated any time new pods created by other schedulers and when they are deleted.

In this project you will contribute to the KubeFlux open-source project by

  • implementing a controller for job cancellation management (for pods allocated by KubeFlux)
  • proposing and implementing methodologies to dynamically update Flux internal graph of resources to address resource sharing between Kubernetes and Flux (for pods allocated by other schedulers)
