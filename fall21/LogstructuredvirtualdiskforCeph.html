Project Overview:

Background:  Remote block devices are widely used in public and private clouds, providing the virtual disks underlying most cloud virtual machines, as well as serving other roles such as Kubernetes Persistent Volume Claims (PVCs).

Ceph (ceph.com) is a widely-used open source scale-out storage system, comprising a RADOS object store with block, file, and S3-compatible interface endpoints. Ceph and "RBD" — its RADOS block device

(https://docs.ceph.com/en/latest/architecture/#ceph-block-device) — are widely used in public and private clouds.

Project Specifics: A Northeastern/BU research team has developed a next-generation log-structured remote block device with much higher performance due to improved write efficiency and client-side SSD caching. Currently this block device supports a Linux kernel interface, which does not work well with virtualized deployments (generally handled by QEMU emulated I/O devices) or Kubernetes PVCs, which use a solution based on Network Block Device.

This project will modify the research system (written in Go) to expose itself via the same interface as librbd, the user-space library that both QEMU (https://docs.ceph.com/en/mimic/rbd/qemu-rbd/) (OpenStack) and rbd-nbd (https://docs.ceph.com/en/mimic/man/8/rbd-nbd/) (Kubernetes) use to access RBD virtual disk images. The core project requirement will be exposing Go functions and hooking them up into a C-compatible interface.

Using either LD_PRELOAD (a standard mechanism to substitute compatible libraries) or patches directly integrating these calls into librbd, this will allow running virtual machines and containers using the new block device without large changes to the associated infrastructure.

Basic  project goals include:

  • A library implementing the basic librbd function interfaces (C-compatible) which the Go-language research block device
  • Demonstrating a QEMU/KVM virtual machine can operate over the virtual block device with this library

Stretch goals could include:

  • additional tools for image creation, etc.
  • demonstration of using the research virtual disk via QEMU/KVM in the OpenStack framework
  • demonstration of Kubernetes PVCs based on research disk and rbd-nbd
  • running existing RBD test suites over research disk
  • modifications and improvements to the research disk
