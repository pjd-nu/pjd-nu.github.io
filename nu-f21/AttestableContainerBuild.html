Project Overview:

Background:  Container image build process is arguably the most critical stage in the application build. During a typical build process, we download and install dependencies, compile our code and finally package it into an image and distribute it. For supply chain security guarantee, it is extremely critical to be able to provide attestation for every build step. In this project, we will design and develop a "build monitor" using eBPF to introspect network connections, file accesses happening during a build process. And we will provide signed and attestable records for every step performed during the "build".

 

Project Specifics: Container image build recipe is typically defined in "Dockerfile". Each step in the Dockerfile could perform one or more actions including installing dependency, copying file, adding user, etc. Each of these actions need to be auditable for provenance record. So in this project, we will generate an in-toto provenance record for the build process that would map to the final image (every layer in the image, to be specific). This would help users of these images to verify every single step  on how that image was built.
