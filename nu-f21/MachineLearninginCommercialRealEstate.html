Project Overview:

Background:  

Commercial real estate has been evolving from a pen-and-paper industry to one where data is more easily accessible. Many cities are now providing real estate transaction information online for transparency and accountability, though the data is still not being used to its full potential.

BLDUP is a Commercial Real Estate SaaS platform that tracks construction through its entire lifecycle by programmatically collecting data from many disparate sources to track construction projects throughout their lifecycle. Our clients can use this data to learn about construction going on in their area of interest and identify leads for their business. The company is still in its startup phase, but has gained significant traction in the Boston area in the last couple of years and is in the process of expanding nationwide.

We’ve built a robust pipeline that helps clean, validate, and normalize data we gather to provide a uniform experience across different geographical areas and data sources. Our dataset is in great shape to train state-of-the-art machine learning algorithms that can help augment the data we provide.

Project Specifics:

For this project, your team will work with the CTO and other team members to identify, train, and deploy useful machine learning models on BLDUP’s dataset. The goal will be to deploy the models as production-quality microservices using the latest machine learning technologies and cloud services to be used as part of the platform.

We’ve identified a number of great machine learning use cases that can be built with our data, including natural language generation of project descriptions and ranking a list of sales opportunities based on a company’s specific interests.

Depending on the team’s strengths, we will identify the models to pursue during initial brainstorming sessions and one (or ideally some/all) of these use cases can be tackled over the course of the project, with each model being trained, tweaked, and deployed as an endpoint of an API that can be called repeatedly. Enhancements, such as a caching layer (where relevant) and an authentication/authorization layer can also be added to make the service more easily usable in production.

A more in-depth example of a machine learning model implementation specific to our platform is the classification of building permits (construction licenses issued by a city). Many permit data sources offer a description of the work as well as the permit classification (demolition, foundation, electrical, plumbing, etc.) and we use these classification types to automate status updates site-wide. However, some data sources only provide partial listings and may not include a classification. With a model trained on the existing mappings of description to classification, it should be possible to accurately predict permit classifications for data sources that don’t provide them to allow the platform to achieve a greater level of automation.

Another example of a model that can be built using our data is a thumbnail/image generator based on the data available for a specific site (e.g. brick 4-story building with a rubber roof) by training a generative adversarial network using existing data and images.

It is an ambitious project that will give the team real-world experience building and deploying production-quality code in a startup environment. We have several team members with machine learning experience that can help discuss/guide the research and development throughout the semester.
