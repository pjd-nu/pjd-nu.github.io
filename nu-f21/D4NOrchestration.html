Project Overview:

Background:  Ceph is the most popular open source distributed storage solution used by the majority of private

 clouds and a number of public clouds. A collaboration between researchers at BU, NEU, and Red Hat has been developing D4N, a data center wide cache integrated into Ceph's Rados Gateway (RGW).  These modified RGW servers can be distributed around the datacenter, enabling data to be cached in solid state storage (SSDs) near computer clusters to reduce load on the data center network and improve performance.  The RGW/D4N caches use a distributed directory, implemented using Redis, to enable requests to be handled from cached state anywhere in the data center.  Also, D4N implements a write-back cache, implemented by using a Ceph storage cluster on SSD storage that is distributed across the RGW/D4N servers.

The D4N work has been going on in a research branch of the main Ceph repository.  This project will take on key efforts to start integrating the software into the upstream open source repositories and make it available to the broader Ceph community.  In particular, this project team will focus on integrating the needed components of D4N into CEPHADM, the orchestration system for Ceph that automates its deployment.  

Project Specifics: While tasks will evolve in an agile way as the project progresses, they may include:

  • Automating the deployment of D3N (the predecessor to D4N that has already been upstreamed.
  • Automating the deployment of the Redis directory used by D4N to record location of cached entries.
  • Automating configuration of a cache and capacity pool of storage.
  • Automating setting up and integrating new RGW instances for elasticity, and new write-back caches with their own OSD deployments.
  • Documenting tasks
  • Developing unit tests for all automation changes.
